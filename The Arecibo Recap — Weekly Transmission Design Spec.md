# **The Arecibo Recap — Weekly Transmission Design Spec**

**Addendum to PK Final Form PRD** **Date:** March 22, 2026

---

## **The Concept**

In 1974, humanity sent the Arecibo Message into space — a pixelated bitmap transmission saying: "We exist. Here's what we're made of." It had 7 sections, each encoding something fundamental about who we are and where we come from.

In Payday Kingdom, the agent does the same thing back to the user.

At the end of each week, the agent composes an **Arecibo Recap** — a pixelated, color-coded 7-section transmission that says: "I see you. Here's what your week was made of." Same format. Same 73×23 pixel grid aesthetic. But the content is personal, generated from the Fact Layer and Taste Layer, with bounded creative freedom in how each section is expressed.

**The thematic parallel:** Humanity introduced itself to the cosmos. Your agent introduces your week back to you.

---

## **Why This Works**

**It's nerdy and beautiful at the same time.** The Arecibo Message is one of the most iconic images in science history. Nerds recognize it instantly. Non-nerds think it looks cool and mysterious. Both groups screenshot it.

**It's inherently shareable.** A vertical, color-coded pixel grid with personal data encoded into it is visually striking on every social platform. It reads as "I got something special" not "here's my spreadsheet."

**It's never the same twice.** Each of the 7 sections has derivative options — multiple visual representations per category. The combinatorial explosion means two identical recaps are astronomically unlikely, even for users with similar weeks.

**It's on-brand for the voxel aesthetic.** PK already lives in pixel art. The Arecibo grid IS pixel art. It's not a stretch — it's a natural extension of the visual language.

**It honors the Bond.** At Bond Level 1, the recap is generic. At Bond Level 5, it's deeply personal — the agent encodes your favorite symbols, references your stated inspirations, and uses your preferred color palette. The recap gets more "you" the more the agent knows.

## **Section Details \+ Derivative Options**

Each section has a base visual representation AND multiple derivatives the agent can choose from. This is what makes every recap unique.

### **Section 1: THE COUNT (Color: White)**

*Original: Numbers 1-10 — the quantitative foundation*

**What it encodes:** The raw numbers of your week.

**Base data points:**

* Bills paid / total bills  
* Habits completed / total check-ins  
* Meetings survived  
* Days active (logged at least one fact)  
* Bond XP earned this week

**Visual representation:** Pixelated number glyphs arranged in the Arecibo counting style (binary-inspired columns, read top-to-bottom).

**Derivatives (agent chooses based on what matters most this week):**

* Standard count (all 5 stats)  
* Highlight mode (one standout stat gets enlarged, others shrink)  
* Streak emphasis (if a streak is active, the count pulses or glows)  
* Minimal mode (only 2-3 most meaningful stats, rest omitted for clean look)

---

### **Section 2: THE ELEMENTS (Color: Purple)**

*Original: Atomic numbers of H, C, N, O, P — the building blocks of DNA*

**What it encodes:** The core habits/routines that formed the foundation of the week. The "elements" your week was made of.

**Base data:** Active habits with completion rates.

**Visual representation:** Each habit is a pixelated "element symbol" — a small glyph in the grid. Completed habits are solid. Missed habits are hollow/dimmed.

**Derivatives:**

* Full element display (all habits shown)  
* Dominant element (highest-streak habit gets a larger glyph)  
* New element discovery (if a new habit was added this week, it gets a "discovery" highlight)  
* Element decay (if a habit streak broke, the glyph shows a crack/fade)

---

### **Section 3: THE PATTERN (Color: Green)**

*Original: Nucleotide formulas — how building blocks combine*

**What it encodes:** The rhythm of the week — how busy vs. calm each day was.

**Base data:** Daily intensity score derived from meetings \+ habits \+ milestones logged per day.

**Visual representation:** A 7-column mini-heatmap where each column represents a day (Mon-Sun). Column height or density reflects intensity. Looks like a waveform or frequency readout.

**Derivatives:**

* Heatmap bars (taller \= busier)  
* Waveform (smooth curves connecting daily intensities)  
* Binary pulse (each day is a dense/sparse block pattern, like actual Arecibo encoding)  
* Bookend emphasis (highlights start and end of week contrast — "started rough, ended strong")

---

### **Section 4: THE THREAD (Color: Blue/White)**

*Original: Double helix \+ genome size — the code of life*

**What it encodes:** The running narrative — active streaks, milestone progress, ongoing journeys.

**Base data:** Longest active streak, total consecutive weeks engaged, milestone proximity (e.g., "3 days from 30-day streak"), cumulative kingdom growth.

**Visual representation:** A pixelated double-helix-style spiral or vertical thread, with markers at significant points. Length represents continuity. Markers represent milestones hit or approaching.

**Derivatives:**

* Clean helix (simple spiral with milestone dots)  
* Growth thread (helix gets thicker/denser toward bottom, showing progression)  
* Near-miss thread (if a milestone was almost reached, a pulsing node appears near the top)  
* Origin thread (if this is Cycle 1 / early weeks, shows the "genesis" — small, but growing)

---

### **Section 5: THE REFLECTION (Color: Red)**

*Original: Human figure with height and population — who we are*

**What it encodes:** How the agent sees you this week. A character portrait/mood representation.

**Base data:** Overall week sentiment (derived from wins vs losses, habit completion rate, hard-week flag), Bond Level, dominant mood of the week.

**Visual representation:** A pixelated figure (echoing the iconic Arecibo stick figure) but styled as the companion — in a pose that reflects the week's character.

**Derivatives:**

* Victory pose (arms up, sword raised — strong week)  
* Working pose (bent over desk/anvil — grind week)  
* Resting pose (seated, contemplative — recovery week)  
* Guardian pose (shield forward — defensive/survival week)  
* Celebration pose (dancing, confetti pixels — milestone week)  
* Vigil pose (standing by lantern — hard week, dignified)

**Bond Level enhancement:** At higher Bond Levels, the figure holds items related to the user's stated interests (guitar, book, paintbrush, etc.) instead of generic sword/shield.

---

### **Section 6: THE KINGDOM (Color: Yellow)**

*Original: Solar System highlighting Earth — where we are*

**What it encodes:** The state of your world — a miniature snapshot of the current kingdom/diorama.

**Base data:** Island stage, number of permanent decorations, number of trophies, active mood theme.

**Visual representation:** A tiny top-down pixel map of the island/tower, with dots representing key structures. Highlighted elements show what was added or changed this week.

**Derivatives:**

* Overview map (all structures visible, new additions glow)  
* Growth comparison (this week vs last week side-by-side, showing what expanded)  
* Spotlight (zoomed into one specific new addition with detail)  
* Seasonal variant (map colored by the active mood pack — cozy warmth, tactical cool, etc.)

---

### **Section 7: THE SIGNAL (Color: Cyan/Teal)**

*Original: Arecibo telescope — how we reached out*

**What it encodes:** The agent's personal message to you. The most creative, most Bond-influenced section.

**Base data:** Full context of the week \+ Taste Layer preferences \+ Bond Level.

**Visual representation:** A pixelated "transmission dish" shape (echoing the Arecibo telescope) with a message area. The message is the agent's weekly line — a short, personal phrase.

**Derivatives:**

* Fact-grounded encouragement ("4 meetings. 3 habits. The streak holds.")  
* Quote transmission (a quote from the user's stated favorite artist/thinker, rendered as a decoded "signal")  
* Verse transmission (if faith mode is on, a short verse rendered as a signal)  
* Symbolic message (no text — just a symbol the user loves, rendered large in the transmission area: a bird, a lantern, a star)  
* Callback message (references something from a previous week: "Remember Week 3? You're still building on it.")

**Bond Level gating:**

* Level 1-2: Generic encouragement only  
* Level 3: Can use stated symbols  
* Level 4: Can quote stated sources  
* Level 5: Can do callbacks, deep personalization, symbolic-only messages

## **Combinatorial Uniqueness**

With derivatives per section, the number of possible unique recaps is:

Section 1: 4 derivatives  
Section 2: 4 derivatives  
Section 3: 4 derivatives  
Section 4: 4 derivatives  
Section 5: 6 derivatives  
Section 6: 4 derivatives  
Section 7: 5 derivatives

Total combinations: 4 × 4 × 4 × 4 × 6 × 4 × 5 \= 30,720 unique visual compositions

And that's before factoring in the actual data values, colors from the Taste Layer, and Bond Level variations. In practice, no two users will ever receive the same recap. Even the same user will almost never see the same composition twice.

## **Rendering Spec**

### **Grid Dimensions**

* **73 rows × 23 columns** (matching the original Arecibo Message exactly)  
* Each cell is a single pixel in the bitmap  
* Rendered at display size as a crisp pixel-art grid (no anti-aliasing, nearest-neighbor scaling)

### **Color Coding**

Each section gets a distinct color, matching the traditional Arecibo Message color-coding convention used in educational materials:

Section 1 — THE COUNT:      White (\#FFFFFF)  
Section 2 — THE ELEMENTS:   Purple (\#9B59B6)  
Section 3 — THE PATTERN:    Green (\#2ECC71)  
Section 4 — THE THREAD:     Blue (\#3498DB) \+ White  
Section 5 — THE REFLECTION: Red (\#E74C3C)  
Section 6 — THE KINGDOM:    Yellow (\#F1C40F)  
Section 7 — THE SIGNAL:     Teal (\#1ABC9C)  
Background:                  Black (\#000000)

**Bond Level color override:** At Bond Level 3+, colors shift toward the user's stated color family preference. The structure stays the same but the palette becomes personal.

### **Rendering Pipeline**

1\. Expression Engine assembles week context  
2\. Per-section derivative selection (LLM or template)  
3\. Section data → pixel grid mapping (deterministic per derivative type)  
4\. Grid assembly (73×23 matrix with color values)  
5\. Render to Canvas 2D (pixel-perfect, no smoothing)  
6\. Export options:  
   \- PNG (high-res, shareable)  
   \- Share card (recap grid \+ kingdom name \+ week number \+ companion name)  
   \- Interactive view (tap a section to see the decoded meaning)

### **The Interactive Decode View**

When the user taps/clicks the Arecibo Recap, each section expands with a human-readable decode — just like educational posters that decode the original message.

The Arecibo Recap replaces the simple "weekly summary card" described in the PRD. It IS the weekly summary — just delivered in a format that's 1000x more shareable and meaningful.

### **LLM Call (1 per week, Sonnet-class)**

System prompt:  
"You are \[companion name\], composing this week's Arecibo Recap for \[user\].  
Select one derivative per section based on the week's facts and the user's taste profile.  
Output strictly typed JSON. Do not invent facts. Choose the derivative that  
best represents the character of this week."

## **Where This Fits in the Build Order**

This slots into **Epic 7** of the PRD build order (Weekly Summary \+ Hard-Week Protocol). The Arecibo Recap IS the weekly summary system — it replaces the simpler text-based recap card with something that's structurally richer and infinitely more shareable.

### **Implementation Steps (within Epic 7):**

1. Build the 73×23 grid renderer (Canvas 2D, pixel-perfect)  
2. Implement the 7 section data assemblers (one function per section)  
3. Build derivative selection logic (LLM intent or template default)  
4. Wire section data → pixel grid mapping (each derivative type has a deterministic pixel layout function)  
5. Build the interactive decode overlay (tap section → expand explanation)  
6. Build the share card generator (grid \+ metadata \+ export)  
7. Wire into the weekly loop trigger (end of week → assemble → render → present)
